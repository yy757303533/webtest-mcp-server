# 子流程 A：需求文档 → Excel 用例

## 使用方式

**用例内容全部使用中文**，包括 title、precondition、steps、expected、tags 字段。

用户指定项目名和需求文档路径即可触发：
```
给 huudi 项目生成用例，需求文档在 /path/to/specs.md
给 demo 项目生成用例，需求文档在 /path/to/specs.txt
```

**参数规则：**
- `project`：用户指定的项目名，对应 `projects/` 下的目录名
- 需求文档：用户指定的路径，直接读取文件内容
- 输出文件名：默认 `{项目名}_cases_v1.xlsx`，用户可指定
- 用例编号前缀：项目名大写 + `-T`，如 huudi → `HUUDI-T`，demo → `DEMO-T`

**禁止调用 list_projects_tool，直接使用用户指定的项目名。**

逐章节分析需求文档，**识别每个功能模块和字段类型，按下方规则生成完整用例集**。

## 用例字段规范
```
case_id:       前缀-T四位编号（全局唯一，连续递增）
module:        对应需求章节名称，如"用户注册和登录"
test_type:     功能测试 / 输入校验 / 边界值 / 状态机 / 安全性测试 /
               单选项测试 / CTA按钮测试 / 条件显示测试 / 多步骤流程测试
title:         字段-场景-条件，如"Email-无@符号拒绝"
precondition:  P1. xxx\nP2. xxx（\n 分隔，如无则留空）
steps:         S1. xxx\nS2. xxx（动词开头：点击、输入、留空、选择、勾选）
expected:      E1. xxx\nE2. xxx（具体可验证，见下方规范）
priority:      高 / 中 / 低
tags:          逗号分隔，如"输入校验,必填,邮箱"
```

## 预期结果写法规范（重要）

**必须写具体内容，禁止笼统描述：**

| ❌ 禁止写法 | ✅ 正确写法 |
|------------|------------|
| 注册失败 | 显示错误提示："该字段为必填项" |
| 操作成功 | 跳转到投资详情页，URL 变为 /investment/xxx |
| 显示错误 | 「Email」输入框显示红色边框错误状态 |
| 验证通过 | 表单验证通过，「下一步」按钮从禁用变为可点击 |
| 数据保存 | 金额 100€ 被保存，页面刷新后仍显示 100 |

**预期结果分类：**
- UI 变化：`「字段名」显示红色边框错误状态`
- 错误提示：`显示错误提示："具体文字"`
- 状态变化：`「下一步」按钮从灰色禁用变为高亮可点击`
- 数据保存：`数值 X 被保存，刷新后仍显示`
- 页面跳转：`跳转到 XXX 页面，URL 变为 /path`

## Step 3：一次性写入

**构建完所有用例后，先写 JSON 文件，再调 Python 写入 Excel，不要分批，不要先试1条。**
```bash
cat > /mnt/repos/webtest-mcp-server/projects/{项目名}/cases_temp.json << 'ENDJSON'
[
  {
    "case_id": "HUUDI-T0001",
    "module": "用户注册和登录",
    "test_type": "输入校验",
    "title": "Email-空值拒绝",
    "precondition": "P1. 用户在注册页面\nP2. 页面已完全加载",
    "steps": "S1. 将「Email」留空\nS2. 点击提交按钮",
    "expected": "E1. 空值输入被拒绝\nE2. 「Email」输入框显示红色边框错误状态\nE3. 显示错误提示：\"该字段为必填项\"\nE4. 无法提交表单",
    "priority": "高",
    "tags": "输入校验,必填,空值"
  }
]
ENDJSON
```
```bash
cd /mnt/repos/webtest-mcp-server && python3 -c "
import asyncio, json, sys, os
sys.path.insert(0, 'src')
os.environ['WEBTEST_PROJECTS_DIR'] = 'projects'
from webtest_mcp.server import generate_cases

cases = json.load(open('projects/{项目名}/cases_temp.json', encoding='utf-8'))
result = asyncio.run(generate_cases(
    project='{项目名}',
    cases=cases,
    output_filename='{项目名}_cases_v1.xlsx'
))
print(result)
"
```

看到 `"success": true` 后删除临时文件：
```bash
rm /mnt/repos/webtest-mcp-server/projects/{项目名}/cases_temp.json
```
